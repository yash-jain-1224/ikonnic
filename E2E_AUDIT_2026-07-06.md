## 4. Issues Found

### 4.1 Sitemap Wrong Domain (SEO impact — non-blocking)

**Status:** Needs fix  
**Impact:** Search engines see `ikonnic.vercel.app` URLs instead of `www.ikonnic.com`

The `NEXT_PUBLIC_APP_URL` environment variable on Vercel is set to `https://ikonnic.vercel.app`. The `sitemap.ts` and `robots.ts` use this to generate URLs.

**Fix:** Update the Vercel environment variable:
```
NEXT_PUBLIC_APP_URL=https://www.ikonnic.com
```

The code already falls back to `https://ikonnic.com` if unset, but the env var overrides it.

### 4.2 Cancellation Email Intermittent Failure (non-blocking)

**Status:** Transient  
**Impact:** Customer may not receive cancellation notification (order confirmation works)

The order confirmation email was delivered (`SENT`), but the cancellation email immediately after failed (`FAILED`). This suggests Gmail SMTP rate limiting or a transient auth issue. The notification service has 3 retries with exponential backoff.

**Recommendation:** Monitor; if persistent, consider rotating the Gmail App Password or switching to a dedicated transactional email service (SendGrid/Resend).

### 4.3 Token Not Immediately Revoked on Logout (advisory)

**Status:** By design (stateless JWT)  
**Impact:** Minimal — tokens have short TTL

After logout, the access token remained valid for subsequent requests. This is inherent to stateless JWTs. The refresh token is properly revoked.
